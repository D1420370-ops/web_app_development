# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述
- **背景與動機**：大學生通常生活費有限，需要一個簡單直覺的系統來追蹤每天的開銷與收入，有效管理個人財務，確保月底不會透支。
- **目標用戶**：大學生。
- **核心價值主張**：提供一個輕量、美觀且容易操作的介面，讓大學生能零門檻地養成每日記帳習慣，並透過數據圖表迅速掌握收支痛點與當月可用餘額。

## 2. 功能需求

本系統具備至少 5 項主要功能，對應的使用者故事如下：

1. **紀錄支出**
   - 作為 大學生，我希望 能夠快速輸入金額、選擇類別並加上簡單備註來紀錄每筆支出，以便 追蹤我的花費去向。
2. **紀錄收入**
   - 作為 大學生，我希望 能夠紀錄生活費、打工薪水等各類收入，以便 知道我這個月總共有多少資金可用。
3. **計算月結餘**
   - 作為 大學生，我希望 系統能自動計算並在首頁顯示每月的「總收入減去總支出」，以便 隨時知道這個月還剩下多少預算。
4. **統計支出**
   - 作為 大學生，我希望 能透過分類圓餅圖或圖表查看每月的支出統計，以便 清楚了解我哪一項開銷最大（如：餐飲、交通、娛樂）。
5. **統計收入**
   - 作為 大學生，我希望 能查看收入來源的統計比例，以便 了解我每個月的生活費結構為何。

## 3. 非功能需求
- **技術限制**：
  - 後端網頁框架：使用 Flask (Python) 
  - 前端技術：HTML + Vanilla CSS + JavaScript 以及 Jinja2 樣板引擎渲染
  - 資料庫：使用 SQLite 作為輕量級關聯式資料庫
- **效能考量**：
  - 網頁需具備響應式設計 (RWD)，支援手機版開啟操作，因為大學生大多使用手機記帳。
  - 確保輸入收支紀錄的流暢度與低延遲載入。
- **安全考量**：
  - 應防範基礎的安全漏洞，如確保系統抵禦 SQL Injection (透過 ORM 或 SQLite 參數化查詢) 與防止 XSS 攻擊 (Jinja2 預設跳脫)。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**：
  - 紀錄支出（金額、日期、類別、備註）
  - 紀錄收入（金額、日期、類別、備註）
  - 首頁儀表板（顯示當月總收入、總支出與本月結餘）
- **Should Have (應該有)**：
  - 統計支出與收入的簡易分類圖表（使用簡單圖表套件如 Chart.js 等）
  - 提供檢視歷史紀錄清單，並可進行刪除或修改
- **Nice to Have (加分項)**：
  - 月度預算警告（當結餘低於特定金額時出現紅字提醒）
  - 建立自訂收支類別
