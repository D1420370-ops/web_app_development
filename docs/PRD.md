# 任務管理系統產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：學生在求學期間常需要同時處理多項作業、考試準備及課外活動，容易遺忘重要事項或錯過死線。因此需要一個簡單易用的系統來幫助學生記錄與追蹤待辦事項，並在時間接近時給予提醒。
- **目標用戶**：學生
- **核心價值主張**：提供一個直覺且高效的任務管理平台，讓學生輕鬆新增、編輯待辦事項，並透過提醒功能降低錯過重要期限的風險。

## 2. 功能需求
1. **新增任務**
   - 使用者故事：作為一名學生，我希望能夠新增任務（包含標題、描述、到期日與時間），以便記錄我需要完成的事項。
2. **編輯任務**
   - 使用者故事：作為一名學生，我希望能夠編輯已建立的任務內容，以便在任務細節或期限變更時進行更新。
3. **刪除任務**
   - 使用者故事：作為一名學生，我希望能夠刪除不再需要或誤建的任務，以便保持任務清單的整潔。
4. **標記任務完成**
   - 使用者故事：作為一名學生，我希望能夠將已做完的任務標記為完成，以便清楚追蹤我的學習與工作進度。
5. **任務到期提醒**
   - 使用者故事：作為一名學生，我希望在任務到期時間接近時能收到系統提醒，以便我能及時處理並不會錯過任何重要的死線。

## 3. 非功能需求
- **技術限制**：系統主要架構預設使用 Flask 作為後端框架，搭配 Jinja2 進行頁面模板渲染，資料庫則使用 SQLite 進行輕量級資料儲存。
- **效能與安全考量**：
  - 系統頁面載入時間應保持快速，操作需具備流暢性以確保良好的使用者體驗。
  - 需實作基本的資訊安全防護措施（例如防範 SQL Injection 和 XSS 攻擊），確保學生紀錄的任務資料安全不被惡意竄改。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 任務的基本 CRUD 操作（新增、檢視、編輯、刪除）。
  - 任務完成狀態的切換（標記完成/未完成）。
  - 能夠設定任務的到期日與時間。
- **Should Have (應該有)**：
  - 在畫面上透過顏色或排序（例如依到期日排序）來提示即將到期的任務。
- **Nice to Have (可以有)**：
  - 自動化的推播通知或信件提醒機制（任務接近到期時）。
  - 任務的分類與標籤功能。
